## Before/After Examples

### Example 1: Solution Section

**BEFORE:**
> Step 1: Data Collection
> System pulls all work orders completed that day and extracts technician notes, time logs, materials used, and customer sign-offs.
>
> Step 2: Compliance Scoring
> Each note is scored against a checklist...

**AFTER:**
> Create Visibility
> Every evening, the system reviews each work order completed that day and scores it against a compliance checklist: time stamps logged, contact person documented, work performed described, materials used listed, customer sign-off captured.
>
> Build Accountability
> Technicians with incomplete notes get immediate, specific feedback on what's missing and how to fix it. Not vague "document better" conversations. Just: "Job #4721 is missing materials used—here's the link to add it."

### Example 2: Why It Works

**BEFORE:**
> Immediate Feedback
> Technicians know within 2 hours if their documentation was incomplete.

**AFTER:**
> Visibility Creates Awareness
> When technicians can see exactly what's missing—in real-time, not weeks later—they fix it. The system creates visibility into a problem that was previously invisible until it cost you money.

### Example 3: Key Takeaway

**BEFORE:**
> "Integrate with existing tools - don't add steps, add automation"

**AFTER:**
> "The tool doesn't matter - visibility, accountability, and incentives do"

## How to Use the Writing Guide

### For New Case Studies:
1. **Read the guide** (especially Core Principles and Tone sections)
2. **Interview the client** using provided question list
3. **Fill in the templates** with their actual words
4. **Edit for principles** - search for technical terms and remove them
5. **Run through checklist** before publishing

### For Editing Existing Case Studies:
1. **Search for red flags:**
   - Pricing mentions
   - Technical terms (API, Python, platform names)
   - Vague claims ("significant", "better", "improved")
   - Corporate speak ("leverage", "synergize")

2. **Replace with:**
   - Principle-focused language (visibility, accountability, incentives)
   - Specific ranges ("six figures", "90%+", "5-10 hours")
   - Plain English
   - Business outcomes

3. **Run the "Would I believe this?" test**
   - If it sounds like marketing, rewrite it
   - If it sounds like a tech demo, focus on outcomes
   - If it sounds vague, add specifics (but not fabricated ones)

**Q: "Should we remove the video?"**

A: Apply same principles when creating new ones. Existing short ones can stay as-is.
