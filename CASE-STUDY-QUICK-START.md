## 30-Day Production Schedule

### Week 1: Research & Outline (5 hours)

**Day 1-2: Keyword Research (2 hours)**
- [ ] Use Ahrefs or Google Keyword Planner to research:
  - "invoice write-offs"
  - "billing disputes service business"
  - "field technician documentation"
  - "service company losing money"
- [ ] Make a list of 10-15 keyword variations
- [ ] Note search volume and difficulty for each
- [ ] Choose 1 primary + 5-8 secondary keywords

**Day 3: Client Interview (1.5 hours)**
- [ ] Schedule call with the operations manager from this case study
- [ ] Use the "Questions to Answer Before Writing" from strategy doc
- [ ] Record the call (with permission)
- [ ] Focus on getting their exact words for quotes

**Day 4-5: Outline & Structure (1.5 hours)**
- [ ] Open `CASE-STUDY-TEMPLATE.md`
- [ ] Create your own outline following the same structure
- [ ] Fill in bullet points for each section
- [ ] Map where keywords will appear naturally
- [ ] Note where client quotes will go

### Week 2: First Draft (8 hours)

**Focus: Get words on the page. Don't edit yet.**

**Day 1: Problem Section (3 hours)**
- [ ] Write "The Problem" section (800-1,000 words)
- [ ] Include the $200K+ write-off discovery
- [ ] Describe daily operational pain
- [ ] Explain why previous solutions failed
- [ ] Use client's actual language from interview

**Day 2: Solution Section (2 hours)**
- [ ] Write "The Solution" section (600-800 words)
- [ ] Step-by-step how the system works
- [ ] Include compliance checklist details
- [ ] Explain Slack notification system
- [ ] Describe quarterly bonus structure

**Day 3: Results Section (2 hours)**
- [ ] Write "The Results" section (400-600 words)
- [ ] Lead with big numbers ($200K, 5+ hours saved, 90% compliance)
- [ ] Include 90-day timeline
- [ ] Add client quote about team's reaction
- [ ] Show sustained results (not just one-time)

**Day 4: FAQ Section (1 hour)**
- [ ] Write FAQ section (500-700 words)
- [ ] Use H3 tags formatted as questions
- [ ] Answer every objection a skeptical prospect would have
- [ ] Include pricing transparency
- [ ] Make each answer 75-100 words

### Week 3: Refinement & Optimization (6 hours)

**Day 1: Opening & Context Sections (2 hours)**
- [ ] Write compelling opening hook (150 words)
- [ ] Write "Why This Matters for Service Businesses" section
- [ ] Write "Key Takeaways" section
- [ ] Write CTA section

**Day 2: SEO Optimization (2 hours)**
- [ ] Add primary keyword to H1, first paragraph, and throughout naturally
- [ ] Distribute secondary keywords across H2/H3 tags
- [ ] Add Toronto/GTA/Ontario mentions 4-6 times
- [ ] Write meta title (60-68 chars)
- [ ] Write meta description (155-160 chars)
- [ ] Add 4-6 internal links to other pages
- [ ] Add 2-3 external links to authority sources

**Day 3: Visual Elements (2 hours)**
- [ ] Add 2-3 pull quote callout boxes
- [ ] Create before/after metrics table
- [ ] Add table of contents with anchor links
- [ ] Create custom featured image (or hire designer)
- [ ] Add alt text to all images
- [ ] Choose 1-2 screenshots if available (redact sensitive info)

### Week 4: Technical & Launch (5 hours)

**Day 1: Client Review (1 hour)**
- [ ] Send draft to client for fact-checking
- [ ] Get written permission to publish (even if anonymized)
- [ ] Incorporate their feedback
- [ ] Get final approval

**Day 2-3: Technical SEO (2 hours)**
- [ ] Add Article schema markup (JSON-LD)
- [ ] Add FAQ schema markup
- [ ] Add Organization schema
- [ ] Double-check all internal links work
- [ ] Test page load speed
- [ ] Ensure mobile-responsive
- [ ] Add social sharing meta tags (Open Graph, Twitter Card)

**Day 4: Publish & Distribute (2 hours)**
- [ ] Publish to `/case-studies/eliminate-invoice-writeoffs-service-business`
- [ ] Submit URL to Google Search Console
- [ ] Share on LinkedIn with pull quote
- [ ] Email to newsletter subscribers
- [ ] Post in relevant industry forums/groups
- [ ] Set up conversion tracking on CTA buttons
- [ ] Add to XML sitemap

**Day 5+: Monitor & Optimize**
- [ ] Check Google Search Console for indexing
- [ ] Monitor initial traffic and time-on-page
- [ ] Note any quick wins (social referrals, email clicks)
- [ ] Set reminder to check rankings in 30 days

## Common Questions

### "Should I write this myself or hire a writer?"

**Write the first one yourself** (or with Claude's help). Here's why:
- You know the client story better than anyone
- You understand the business pain intimately
- Your voice/tone should come through
- You'll learn what works for future case studies

**After 2-3 case studies**, consider hiring if:
- You have a backlog of stories to tell
- You'd rather spend time on client work
- You have budget ($500-1,200 per case study for a good writer)

### "How long does writing actually take?"

**First case study:** 20-25 hours total
- 5 hours: Research & outline
- 8 hours: First draft
- 6 hours: Refinement
- 5 hours: Technical setup & launch

**Subsequent case studies:** 12-15 hours
- You'll have a template and process
- Keyword research gets faster
- You know what works

### "What if the client doesn't want to be identified?"

**You can still publish anonymously:**
- "A GTA-based service company"
- "A mid-sized HVAC contractor in Ontario"
- Redact company name, logo, screenshots
- Still use real metrics (they're what matter)

Most clients are fine with anonymized case studies. Get written permission either way.

### "Do I need to keep the video version?"

**Yes! Use both:**
- Long-form written version for SEO and organic traffic
- Video embedded in the written case study for credibility
- Video on YouTube separately for video search traffic
- They serve different purposes and audiences

### "What if I don't have all the metrics?"

**Go back and get them:**
- Schedule a follow-up call with the client
- Pull data from their systems (with permission)
- Estimate conservatively if exact numbers aren't available
- Use ranges ("$150K-$200K" instead of "$200K")

Specific numbers are what make case studies credible. Worth the extra effort.

### "Should I gate these behind an email capture?"

**No. Here's why:**
- Gated content kills SEO (Google can't index it)
- Reduces social sharing (people won't share if others can't read it)
- Adds friction to your discovery call funnel
- The goal is traffic → discovery call, not traffic → email → nurture → call

Make them public. Your CTA is "book a call" right in the case study.
